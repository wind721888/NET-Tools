# 微软消息队列（MSMQ）

## 文件夹
- `Receiver` - 接收端
- `Sender` - 发送端

## 核心特性与优势
1. 异步通信：发送方将消息放入队列后即可继续执行其他任务，无需等待接收方处理或在线。接收方可以在方便时从队列中获取消息。这提高了应用程序的性能和响应能力。
2. 可靠性与持久性：MSMQ能确保消息的可靠传递。即使目标应用程序暂时未运行、网络中断或系统发生故障，消息也会被存储在队列中（通常持久化到磁盘），待恢复后继续处理，保证消息不丢失。
3. 应用解耦：发送方和接收方不需要紧密耦合或同时运行，提高了系统的灵活性、可维护性和可扩展性。
4. 安全性：基于Windows安全性和加密API（CryptoAPI）提供安全通信，支持消息加密和数字签名。
5. 事务支持：支持事务性消息发送，确保一组消息要么全部成功发送，要么全部不发送，保证了数据的一致性。
6. 优先级与路由：支持设置消息优先级，确保关键消息优先处理；提供灵活的路由机制，适应异构网络环境。
7. 脱机能力：即使接收方暂时离线，发送方也可以继续发送消息到队列，增强了系统的容错能力。

## 主要应用场景
1. 分布式系统：在多台服务器间进行可靠的消息传递。
2. 后台处理：处理耗时任务，如订单处理、日志记录、邮件发送等，避免阻塞主流程。
3. 服务间通信：在微服务架构或SOA（面向服务架构）中，作为服务间解耦的通信方式。
4. 事件驱动架构：响应系统内的各种事件。
5. SQL Server Integration Services (SSIS)：通过消息队列任务在包之间或与其他应用程序协调数据处理和文件传输。

## 核心概念
- 消息（Message）：在应用程序之间传递的数据单元，可以是字符串、对象或二进制数据。
- 队列（Queue）：存储消息的容器。主要类型包括：
  - 公共队列（Public Queue）：在网络中发布，可供网络上的其他计算机访问。
  - 私有队列（Private Queue）：只存在于本地计算机，不能在网络中发布，需要知道完整路径才能访问。
  - 管理队列（Administration Queue）：包含确认消息传递的回执。
  - 响应队列（Response Queue）：包含响应消息。
- 发送方（Producer/Sender）：将消息发送到队列的应用程序。
- 接收方（Consumer/Receiver）：从队列中读取消息并进行处理的应用程序。

## 安装与配置

MSMQ在Windows系统中通常不是默认启用的功能，需要手动开启：

1. 打开“控制面板” > “程序” > “启用或关闭Windows功能”。
2. 找到并勾选“Microsoft Message Queue (MSMQ) Server”相关的选项（包括核心、HTTP支持、Active Directory集成等，根据需要选择）。
3. 完成安装后，可以在“计算机管理” > “服务和应用程序” > “消息队列”中看到相关组件，表示安装成功。
